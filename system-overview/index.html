^ Design Patterns

Heartbeat with Per-Task Cadences
Instead of monolithic scheduled jobs, each sub-task has its own interval. The cron job checks what's due and only executes those tasks. Prevents wasted API calls and keeps scheduled work focused. Inspired by OpenClaw's HEARTBEAT.md pattern.
Execute-Verify-Report (EVR)
Every automated action must be executed, verified (file exists, message delivered), then reported. Prevents the most common agent failure: acknowledging a task without completing it. Inspired by OpenClaw's Standing Orders.
Notification Suppression
Scheduled checks that find nothing actionable stay silent. Only ping when something matters. The morning briefing always sends, but mid-cycle checks suppress empty results. Inspired by OpenClaw's HEARTBEAT_OK.
Temporal Context Loading
Today's and yesterday's daily notes are loaded into session context for continuity. Provides "what was I doing?" awareness without loading the full archive. Inspired by OpenClaw's daily memory files.
Iterative Context Compression
Long sessions use a structured template (Goal, Progress, Decisions, Files, Next Steps, Critical Context) to preserve important information across context compactions. Prevents progressive information loss. Inspired by Hermes Agent's ContextCompressor.
Frozen Snapshot Memory
CLAUDE.md loads once at session start and stays stable for prompt cache efficiency. Live state comes from tool calls (reading files, querying Garmin). Memory writes update disk immediately but don't change the active system prompt.
Intent-Based Routing
Inbound Telegram messages are classified by intent and routed to the appropriate vault location. Calendar events go directly to Google Calendar. Tasks route by urgency. Journal entries go to monthly files. Shopping items go to lists.

! Known Limitations

Session-Only Scheduling
Cron jobs are in-memory and expire after 7 days or on session restart. Need to be recreated manually. Persistent scheduling solution still being researched.
Sleep Before Briefing
Laptop must be awake by 7:57 AM for the morning briefing to fire. macOS sleep/wake scheduling could solve this.
Garmin Sync Dependency
Watch must sync to phone before the morning briefing for sleep data to be available. Trent needs to sync before ~7:50 AM.
Cowork Still Active
Cowork scheduled tasks haven't been disabled yet. Can create duplicate daily notes and conflicting vault edits. Needs to be turned off.
