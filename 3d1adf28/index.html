* Current System State

CORE
Heartbeat System — launchd-driven sub-task scheduler. Each task has its own cadence and last-run timestamp in Tasks/heartbeat.md. File-based MCP channel triggers sessions via .trigger files. Published as open source: claude-heartbeat.
CORE
Execute-Verify-Report (EVR) — Every automated action follows three steps: execute, verify the result exists, report. "Done" without verification is not acceptable. If verification fails, report the failure rather than silently moving on.
CORE
Notification Suppression — Mid-day heartbeat checks (backlog nudge, recurring audit) stay silent when there's nothing actionable. Only morning briefing and evening check-in always send. Reduces notification fatigue.
CORE
Session Snapshots — Every 3 days, structured snapshots are saved to Work/Sessions/ as a long-term memory bridge across autocompact cycles. Delta-only updates preserve running context without repetition.

^ Design Patterns

1. Heartbeat with Per-Task Cadences
launchd fires on schedule, writes a .trigger file, MCP channel delivers to session. Each sub-task has its own interval. Only due tasks execute. Published as open source: claude-heartbeat.
2. Execute-Verify-Report (EVR)
Every automated action must be executed, verified (file exists, message delivered), then reported. Prevents the most common agent failure: acknowledging a task without completing it.
3. Notification Suppression
Scheduled checks that find nothing actionable stay silent. Only ping when something matters. The morning briefing and evening check-in always send, but mid-cycle checks suppress empty results.
4. Temporal Context Loading
Today's and yesterday's daily notes are loaded into session context for continuity. Provides "what was I doing?" awareness without loading the full archive.
5. Iterative Context Compression
Long sessions use a structured template (Goal, Progress, Decisions, Files, Next Steps, Critical Context) to preserve important information across context compactions.
6. Session Snapshots
Every 3 days, structured snapshots are saved to Work/Sessions/ as a long-term memory bridge. Delta-only updates capture what was built/changed, key decisions, open items, and running context (health trends, project states).
7. Frozen Snapshot Memory
CLAUDE.md loads once at session start and stays stable for prompt cache efficiency. Live state comes from tool calls (reading files, querying Garmin). Memory writes update disk immediately but don't change the active system prompt.
8. Intent-Based Routing
Inbound Telegram messages are classified by intent and routed to the appropriate vault location. Calendar events go directly to Google Calendar API. Tasks route by urgency. Journal entries go to monthly files. Shopping items go to lists.
9. Webhook Action Registry
External webhook triggers are validated via HMAC signature and checked against an action registry whitelist. No arbitrary text injection — only pre-defined actions are accepted. Cloudflare tunnel provides HTTPS ingress.

! Known Limitations

Cloudflare Quick Tunnel
Quick tunnel URL changes on every restart. Needs migration to a named tunnel for a permanent, stable webhook URL.
Dashboard Webhook Secret
Webhook secret is embedded in dashboard JS. Acceptable for a private randomized URL, but not production-grade. Would need server-side proxy for proper security.
Cowork Still Active
Cowork is still running some scheduled tasks. Needs to be fully disabled to prevent duplicate daily notes and conflicting vault edits.
Exercise HR Dysregulation
Running-specific autonomic dysregulation not yet resolved. Resting biometrics (RHR, HRV) are stable, but exercise heart rate still spiking disproportionately during runs.
